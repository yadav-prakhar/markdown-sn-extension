      Alert Types
      

    

    
10 built-in alert styles

    
Use the > [!TYPE] syntax in your Markdown. Each alert renders with a distinct color in ServiceNow.

    

      
    

    

      Example syntax:
      
> [!WARNING]
> This deployment will cause a brief service interruption.
> Estimated downtime: 5 minutes.

    

    

      ✨ Plus: Unlimited custom alerts
      
Create your own alert types with any name, emoji, and color scheme via the extension settings. Import and export configurations to share with your team.

      Get Started
      

    

    
Up and running
in 60 seconds

    

      

        

          
1

          

            
Install from the Chrome Web Store

            
Click the install button or search "Markdown to ServiceNow" in the Chrome Web Store. No sign-in or account required.

          

        

        

          
2

          

            
Pin the extension

            
Click the puzzle icon in Chrome's toolbar → pin "Markdown to ServiceNow" for quick access. Or use the keyboard shortcut ⌘ShiftM.

          

        

        

          
3

          

            
Open any ServiceNow instance

            
Navigate to any incident, change request, or service request. The toolbar appears automatically on journal fields (Work Notes, Comments, Activity).

          

        

        

          
4

          

            
Write Markdown, click Convert

            
Type your update in Markdown directly in the journal field, click the Convert button in the toolbar — done. Or use the popup for standalone conversion.

      Privacy & Security
      

    

    
Your data never leaves your browser

    
Markdown to ServiceNow is 100% local. There are no servers, no accounts, no analytics.

    

      

        🚫
        
No Data Collection

        
No text you convert is ever stored or transmitted. All conversion happens locally in your browser tab — not on any server.

      

      

        🌐
        
No Network Requests

        
The extension makes zero network requests. It works offline. There are no external API calls, analytics endpoints, or CDN dependencies.

      

      

        💾
        
Local Storage Only

        
Your custom alerts and snippets are stored in Chrome's local storage — on your machine only. They never sync to any external system.

      

      

        🔓
        
Open Source

        
Every line of code is publicly auditable on GitHub. No obfuscated code, no minified scripts with hidden functionality. Read it for yourself.

      

      

        🎯
        
Minimal Permissions

        
Requests only the permissions it needs: clipboard write, context menus, active tab, and storage. No broad host permissions beyond ServiceNow domains.

      

      

        🔒
        
XSS Protection

        
The preview modal sanitizes HTML — stripping event handlers and javascript: URIs before rendering. Shadow DOM isolation prevents style leakage.

In the UI they appear with a separate Display Name that can include spaces and mixed case (e.g., "Hot Fix"). The technical name must be lowercase, alphanumeric, and underscores only (max 20 chars).

However, converting twice replaces the snapshot — the restore button will only undo the most recent conversion. If you've converted and already posted the entry, there's no way to recover the original Markdown from the extension (it lives in your browser memory only).
